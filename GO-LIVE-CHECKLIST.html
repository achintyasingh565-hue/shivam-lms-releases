Shivam Enterprises LMS — Go-Live Checklist

A careful path from "installed" to "running the business on it." Version 2.0.3.

1 · Prove it before you rely on it Do first

  • Run in parallel for 1–2 weeks.Keep your current method going and enter the same real loans in both. This builds confidence with your own data, not test data.
  • Check the numbers match.For a few real loans, confirm the EMI, outstanding, and any prepayment/restructure figures agree with your existing records.
  • Test a backup restore — for real.Install on a spare/empty machine, restore from your backup (or sign into cloud), and confirm every record comes back. A backup you've never restored isn't proven yet.

2 · One-time setup on every device Once each

  • Install 2.0.3 as the baseline.On each Mac (the .dmg) and Windows PC (the Setup .exe) from the Releases page. This is what lets future updates arrive automatically.
  • Set up Supabase cloud sync.Run the schema once in Supabase, then sign in inside the app on every device so records sync across machines.
  • Turn on the folder auto-backup.Administration → Backup → choose a folder. Ideally a folder that itself syncs (e.g. Drive/OneDrive) for an off-machine copy.

3 · Ongoing habits Every so often

  • Confirm backups are actually happening.Glance at the backup status now and then; take a manual backup before big changes (bulk edits, year-end).
  • Keep devices signed into the cloud.So a lost or broken machine never means lost records.
  • Publishing an update (when I send one).GitHub Desktop: copy files in → Commit → Push. Then Actions → Run workflow. Installed apps show the update pop-up on their own.

4 · Optional polish Nice to have

  • Code signing.Apple Developer (~$99/yr) + a Windows certificate removes the "damaged" / "unknown publisher" warnings so installs feel fully professional.
  • Tighten minor edge cases.e.g. cap the schedule when an EMI is unrealistically tiny, and firm up non-integer tenures. None affect normal use — ask me anytime.
If something ever looks wrong: don't panic and don't delete anything. Your data lives in three places (the app's local storage, IndexedDB, and the Supabase cloud) plus your folder backups. Restore from the most recent backup or re-sync from the cloud, and message me with what you saw — I can help recover.
